# README Checklist

Use this checklist to ensure each repository README meets hiring expectations.

## Essentials
- [ ] Clear one-sentence project pitch
- [ ] Live demo link(s)
- [ ] Tech stack badges (frontend/backend/db)
- [ ] Screenshots/GIFs of the app
- [ ] Setup instructions (copy-paste friendly)
- [ ] `.env.example` with documented variables
- [ ] API overview or link to docs
- [ ] Testing instructions (`npm test`, etc.)
- [ ] CI badge(s) and passing status
- [ ] License and author/contact

## Quality Signals
- [ ] Architecture overview diagram or section
- [ ] Data schema (if applicable)
- [ ] Performance notes (caching/pagination)
- [ ] Security notes (auth, validation, CORS)
- [ ] Accessibility considerations
- [ ] Mobile responsiveness confirmed

## Nice-to-Have
- [ ] Demo credentials (test account)
- [ ] Roadmap (next features)
- [ ] What I learned / design decisions
- [ ] Known issues / limitations

## Release Readiness
- [ ] Production env vars documented
- [ ] Deploy steps (Vercel/Render/Heroku/etc.)
- [ ] Health check endpoint noted
- [ ] Monitoring/logging notes (optional)
